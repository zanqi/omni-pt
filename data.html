Data plan: one shared, model-agnostic babble eval set

Keep per-model self-probed training data as-is; replace only the eval set with a single fixed, neutrally-labeled dataset reused across every model.

Why
  • SLOT_SNR in babble_data.py is already one fixed band shared by every model — self-probing doesn't widen/shift it per model, it just rejection-samples inside that fixed band until the classifier labels a draw the way it needs.
  • Measured on the actual pushed datasets: 968/1000 train slurp_ids overlap between the Qwen2.5 and Qwen3 babble datasets (3.2% content drift from per-model probe failures), and for the utterances that DO overlap, the realized SNR for the same (slurp_id, kind) differs with std ≈ 5.5dB between the two.
  • Every cross-model C/R/F/EAR comparison made so far (the whole point of eval.ipynb's tables) has been comparing models on different underlying audio, not just different models.
  • Training data stays self-probed on purpose — SFT/DPO should see the specific noise instances that confuse that exact model. Only eval needs to be shared/neutral.

Design: single fixed listener replaces the omni self-probe, eval-only

Today, probe_triplet (in babble_data.py) uses the trained-on omni model itself for two passes per noisy draw: an ASR transcript and a task response ("Heard: ... Reply: ..."), and feeds both to the vLLM classifier as independent survival signals. For the eval set, replace both passes with a single fixed external ASR model — Whisper-large-v3 (via transformers, already available in this stack) — that never changes across runs, and drop the "task response" signal entirely since Whisper doesn't perform tasks, only transcribes.

Steps

1New script: babble_eval_data.py

A sibling to babble_data.py, not a modification of it — the train-data builder must stay untouched. Reuse directly from babble_data.py: make_probe_batch (noise synthesis is already model-agnostic), generate_targets + TARGET_PROMPT (target text generation doesn't depend on the listener), the vLLM client/gpt_json setup, SLOT_SNR/SLOT_WEIGHTS/KINDS constants.

2Load Whisper once, batch-transcribe

from transformers import WhisperForConditionalGeneration, WhisperProcessor

whisper_model = WhisperForConditionalGeneration.from_pretrained(
    "openai/whisper-large-v3", torch_dtype=torch.bfloat16, device_map={"": 0}
)
whisper_processor = WhisperProcessor.from_pretrained("openai/whisper-large-v3")

def whisper_transcribe_batch(audios):
    inputs = whisper_processor(audios, sampling_rate=16000, return_tensors="pt", padding=True)
    inputs = inputs.to(whisper_model.device, dtype=whisper_model.dtype)
    ids = whisper_model.generate(**inputs, language="en", task="transcribe")
    return [t.lower().strip() for t in whisper_processor.batch_decode(ids, skip_special_tokens=True)]

This replaces both base_generate_batch calls (ASR pass + task-response pass) inside probe_triplet with a single call.

3Trim the classifier prompt to transcript-only

CLASSIFY_PROMPT currently frames its input as "two independent passes" (transcript + assistant response) and lets either one prove a piece survived. Whisper only produces a transcript, so write CLASSIFY_PROMPT_FIXED: same rubric (answer / repair / repeat definitions, the "key info" test, the JSON output shape) but with the "two independent passes" framing removed and every survival check based on the transcript alone. _span_survived's post-processing (the deterministic override that trusts a verbatim match) narrows to checking the transcript only, not (transcript, response).

Trade-off to accept: the current two-signal design exists partly because an ASR transcript can be locally wrong on a word while the full utterance is still contextually recoverable (that's what the "response" signal was catching). Dropping it makes the fixed eval set's ground truth strictly bound by Whisper's transcription quality — a Whisper mistranscription now directly mislabels the row, with no second chance. Mitigate by using the largest Whisper checkpoint (large-v3, not turbo) and spot-checking ~20 generated labels by ear before trusting the set.

4Utterance selection & exclusion set

Build the seen_slurp_ids exclusion set from more sources than today's single MASK_DS_ID:

EXCLUDE_DS_IDS = [
    "keylazy/slurp-ear-sft",                                   # EAR track (existing)
    "keylazy/slurp-babble-Qwen2.5-Omni-3B-v3",                 # Qwen2.5 train (avoid eval/train leakage)
    "keylazy/slurp-babble-Qwen3-Omni-30B-A3B-Instruct-v1",     # Qwen3 train (same reason)
]

Every model's own self-probed train split already excludes the EAR set; this adds excluding each OTHER model's train utterances too, so the shared eval set never overlaps anything any model was tuned on. Extend this list whenever a new model's train set is built.

Recommend a larger N than today's 50 per kind (e.g. 150) — more eval rows means less noise when comparing four-plus models on the same set going forward, and this only needs to be built once, ever.

5Push one dataset, no model name in the repo id

REPO_ID = "keylazy/slurp-babble-fixed-eval-v1"   # deliberately no model name

Single test split (no train needed — this dataset is eval-only by design). The naming itself signals "canonical, shared, don't rebuild per model."

6Re-point eval.sh at the shared set

babble_eval_qwen.py needs zero code changes — it already takes --dataset/--split as flags. Just change every invocation in eval.sh:

python babble_eval_qwen.py --model-path Qwen/Qwen2.5-Omni-3B \
    --dataset keylazy/slurp-babble-fixed-eval-v1 --split test ...
python babble_eval_qwen.py --model-path Qwen/Qwen2.5-Omni-3B \
    --adapter-path ./Qwen2.5-Omni-3B-bab-sft-v3 \
    --dataset keylazy/slurp-babble-fixed-eval-v1 --split test ...
python babble_eval_qwen.py --model-path Qwen/Qwen3-Omni-30B-A3B-Instruct \
    --dataset keylazy/slurp-babble-fixed-eval-v1 --split test ...
python babble_eval_qwen.py --model-path Qwen/Qwen3-Omni-30B-A3B-Instruct \
    --adapter-path ./Qwen3-Omni-30B-A3B-Instruct-bab-sft \
    --dataset keylazy/slurp-babble-fixed-eval-v1 --split test ...

7Re-run and expect the numbers to move

All four bab_results_*_slurp.jsonl files get regenerated against the new shared set. Every C/R/F/EAR/confusion-matrix comparison made in eval.ipynb so far was computed on four different eval sets — expect numbers to shift once they're all on the same one, and treat the new numbers (not the old ones) as the trustworthy comparison going forward.

Scope check: what does NOT change

  • babble_data.py (train-set builder) — untouched, keeps self-probing per model.
  • babble_eval_qwen.py (eval runner) — untouched, already dataset-agnostic via --dataset.
  • The EAR/word-masking track — already model-agnostic by construction (deterministic forced-alignment word masking, not self-probed), no issue there to fix.
  • Existing per-model train datasets (keylazy/slurp-babble-Qwen2.5-Omni-3B-v3, ...Qwen3...-v1) — kept as-is for SFT/DPO.
